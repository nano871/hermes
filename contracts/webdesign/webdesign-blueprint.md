# SAFFRON DESIGNS — WEB DESIGN SERVICE CONTRACTS

## Key Differences: Web Design vs. AI Consulting

| Dimension | AI Consulting (Saffron Automations) | Web Design (Saffron Designs) |
|-----------|-------------------------------------|------------------------------|
| **Deliverable** | Custom AI solution, often ongoing | Website — fixed scope, defined end |
| **IP model** | Client owns deliverable; Saffron retains tools | Client owns final site; Saffron retains templates, frameworks, design system |
| **Payment** | T&M or fixed per SOW | Fixed price + milestone payments + deposit |
| **Revisions** | Iterative via SOW updates | Defined revision rounds (typically 2-3) |
| **Third-party assets** | APIs, model licenses | Stock photos, fonts, plugins, hosting |
| **Portfolio rights** | Confidentiality usually restricts | **Essential** — designers need to show work |
| **Timeline** | Flexible, ongoing | Fixed, milestone-based |
| **Client responsibilities** | Data access, decisions | **Content delivery** — biggest risk to timeline |
| **Post-launch** | Ongoing support possible | Maintenance retainer or handoff |
| **AI tools used** | Custom AI development | May use AI for content, images, code generation |

## Document: Web Design Services Agreement

Covers: web design, development, e-commerce, landing pages, redesigns, maintenance

Key clauses specific to web design:
1. **Deposit & Milestones** — 50% deposit to start, milestone-based payments
2. **Client Content Responsibilities** — content delivery deadlines (biggest failure point)
3. **Revision Rounds** — typically 2-3 rounds, additional rounds billed
4. **Portfolio Rights** — designer must be able to show the work
5. **Template Ownership** — Saffron retains template/framework rights
6. **Third-Party Licenses** — stock assets, fonts, plugins pass-through
7. **Browser Compatibility** — defined scope (modern browsers, last 2 versions)
8. **Post-Launch Support** — warranty period vs. maintenance retainer
9. **AI-Generated Content** — disclaimer if AI tools used in design process
10. **Domain & Hosting** — who registers/holds, who pays
